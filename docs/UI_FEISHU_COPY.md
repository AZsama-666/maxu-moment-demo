一、文档说明

1.1 这份文档写什么
严格对齐 up9.life 现有 Demo，只写已实现或占位页面。不含大屏应援、心愿求购、预售、即时在线等未落地 SKU。

1.2 设计优先级
P0：买家 1V1 全链 + 供给发起/履约（先做高保真）
P1：我的 Moment、管理、消息、组局
P2：Feed、Profile 壳层、分类、转约占位、404

1.3 画板与色值
画板：390 × 844
主色占位：#12B89A
禁用表达：市集、NFT、链上、漂流、陪聊、私密视频
Demo 内个别「回到市集」→ 设计稿统一写「回到 Moment」

三、必跑 Demo 路径

3.1 路径 A · 买家 1V1（P0）
Moment 首页 → TA 主页 → Moment 详情 → 选档 → 确认下单 → Mock 支付 → 买家等待室 → 语音/视频履约 → 完成页

操作要点：
① 详情页选时段，点「去下单」
② 支付页点「支付成功」
③ 等待室点「我已就位」→「双方已就位 · 开始履约」（或到点「到点 · 进入履约」）
④ 履约页倒计时结束或点「挂断」→ 完成页

3.2 路径 B · 供给发起 + 履约（P0）
我的 → 我的 Moment → 发起四步 → 待处理任务 → 供给等待室 → 供给履约 → 回待处理任务

操作要点：
① 任务页只有「进入等待室」，不能在这里标记就绪
② 供给等待室内点「标记就绪」
③ 履约结束点「结束服务」，回任务页

3.3 路径 C · 组局 / 陪玩 / 转约（P1）
组局（主理人召集）：Moment 首页「组局」Tab → 组局活动卡 → 组局详情 → 报名下单 → 订单确认 → 双方模拟确认（TA 页组局区块亦可进入）
陪玩（本质 1V1）：全部/1V1 Tab 人卡 → TA 页「陪玩」区块 → 陪玩卡 → 陪玩详情 → 立即下单 → 订单确认 → 双方模拟确认
转约：转约占位页 → 即将开放


四、全局交互规则

4.1 标记就绪
买家：等待室内「我已就位」
供给：供给等待室内「标记就绪」
任务页不出现标记就绪按钮

4.2 提前开始
双方都已就位 → 可提前「双方已就位 · 开始履约」
到点仍可「到点 · 进入履约 / 开始履约」

4.3 Demo 联调
单方点就绪会模拟对方也已就位（方便一个人测两条链路）

4.4 近档退款
距预约 < 60 分钟：到点前 3 分钟供给未就绪，买家可申请退款；到点未履约自动退

4.5 远档确认
页面 UI 已有，Demo 默认不触发

履约中 → 继续履约
已完成 → 查看完成页
